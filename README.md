## Portfolio Reflection

### Interpreting User Needs

One of the biggest takeaways from this course was learning how to take user needs and turn them into software requirements that could actually be implemented. Creating user stories helped break larger requirements into smaller, more manageable tasks while keeping the focus on delivering value to the customer. Throughout the SNHU Travel project, user stories made it easier to prioritize features, organize the product backlog, and ensure development stayed aligned with stakeholder expectations.

### Agile Development

This course gave me a much better understanding of how Agile development works in practice. Instead of trying to complete everything at once, I learned how planning, development, testing, and feedback all work together throughout each sprint. In future projects, I hope to continue using Agile practices such as sprint planning, backlog prioritization, regular communication, iterative development, and continuous improvement to build higher-quality software and respond more effectively to changing requirements.

### Teamwork

To me, being a good software development team member means contributing to the team's goals while also helping others succeed. Throughout this project, I experienced the responsibilities of multiple Scrum roles, which reinforced the importance of communication, accountability, collaboration, and adaptability. I learned that successful software development depends not only on technical skills but also on working effectively with others, sharing feedback, and remaining focused on delivering value to stakeholders.
